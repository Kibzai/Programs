## Git
Git is a version control system. Its like a timelined branched repository with history.

### Commands
> Series of protocols to use Git.
- [x] Init: Creates a Git repository by using: `git init`.
- [x] Remote: Adds a remote repository to the Git repository by using: `git remote add <name> <url>`.
- [x] Status: Shows the status of the Git repository by using: `git status`.
- [x] Add Remote: Adds a remote repository to the Git repository by using: `git remote add <name> <url>`.
- [x] Clone: Clones a Git repository by using: `git clone <url>`.
- [x] Fetch: Fetches changes from a remote repository by using: `git fetch <name>`.
- [x] Checkout: Checks out a different branch or commit by using: `git checkout <branch | commit>`.
- [x] Add: Stages changes in the working directory by using: `git add <file(s)>`.
- [x] Commit: Commits changes in the working directory by using: `git commit -m "<message>"`.
- [x] Delete: Deletes a branch or commit by using: `git branch -d <branch>` or: `git commit --amend`.
- [x] Update: Updates a branch to the latest version by using: `git pull <name> <branch>`.
- [x] Push: Pushes changes to a remote repository by using: `git push <name> <branch>`.
- [x] Pull: Pulls changes from a remote repository by using: `git pull <name> <branch>`.
- [x] Branch: Creates a new branch by using: `git branch <name>`.
- [ ] Pull Request
